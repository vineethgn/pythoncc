Future enhancements

Pythoncc does not yet fully support all Python built-in features including several built-in keywords, dynamic typing, object oriented programming, functional programming, modular programming, etc. All these features need to be implemented so that the compiler is complete.

The current approach of generation of several stages of intermediate representation is unnecessary. The binary code must be generated directly from the LLVM IR in future versions.

Currently, no optimization is done on the code, (except for LLVM internal optimizations). The generated code can be made much more efficient by implementing optimization phases.

Pythoncc can be written as a front-end to GCC so that it can be merged with the GCC development branch.

The front-end can be written in the Python language itself by making use of the Python compiler module. [12]
