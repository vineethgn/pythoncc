Existing system

Currently, all Python scripts are run with the help of a Python interpreter in all environments. The key feature of its performance is the algorithmic efficiency of the interpreter.

It translates the complete script into a byte-code format (.pyc) when the script is loaded. The byte-code is saved for future use. The interpreter is also capable of generating an optimized version of the byte-code (.pyo). [3]

Simple applications seem to work with acceptable amount of performance by this method. However, applications with heavy computational complexity happen to be very slow.

There are other third party programs like py2exe [4] which create MS Windows executable from Python scripts. But what they do is simply fuse the script and the Python interpreter into a single executable file, so that it will appear like a compiled executable program.
