Code generation

The actual code generation is done by LLVM and GCC. After the complete IR code for the input has been generated, the LLVM API is used to generate the LLVM assembly code. This code is compiled using an LLVM tool called llc which generates x86 assembly code. This assembly is finally compiled into a Linux binary using GCC (although, any compatible compiler will work).

An alternative to the LLVM approach is using the GCC code base. GCC has divided the compilation stages into three; the front-end that takes care of lexical and syntax analysis, the middle-end which performs code optimization and finally the back-end for machine code generation. The front-end generates an intermediate representation called GENERIC which is lowered to the middle end. This approach appears to be seamless but, there is a problem of lack of documentation for the GCC code. Most of the documentation as well as examples are out of date.
