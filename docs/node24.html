Statement grouping

In Python, statement blocks are identified from the indentation of the code. Contiguous code written on the same indentation level belongs to the same code group. Nesting of indented statement blocks is also possible.

This is acheived by the use of two tokens, INDENT and DEDENT. The INDENT token is generated at the start of an indented statement block. And at the end, the DEDENT token is returned. The number of DEDENT tokens finally generated should always be equal to the number of INDENT tokens. [10]

This is done by the use of a stack and a special state in the lexical analyzer. The stack keeps track of multiple indentation levels and the special state is used to generate multiple DEDENT tokens whenever necessary.
